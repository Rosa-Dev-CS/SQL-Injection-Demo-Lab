    Understanding Vulnerable Login

    Because this system directly concatenates your inputs into a SQL statement inside app.py:

    query = f"SELECT * FROM users WHERE username = '{username}' AND password = '{password}'"

    The database engine cannot tell the difference between database structure commands and actual text. A single quote (') ends the string value and starts a raw command sequence.

    Try these lab exercises:

    Exercise 1

    Bypass login criteria: type ' OR '1'='1 as the username and anything as password.

    Evaluates to: WHERE username = '' OR '1'='1' (Always TRUE)

    Exercise 2

    Hijack Admin Account: type admin' -- as the username and anything as password.

    Evaluates to: WHERE username = 'admin' --' AND password = '...'

    The double dash (--) comments out the password checks, causing SQLite to only evaluate username = 'admin'.
