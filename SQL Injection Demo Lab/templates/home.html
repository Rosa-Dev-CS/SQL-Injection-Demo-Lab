Vulnerable Node

Insecure Login Page

This system dynamically constructs SQL statements via direct string interpolation. This allows user input to break out of the string boundary and modify the query structure.

Vulnerable Code: "SELECT * FROM users WHERE username = '" + user + "' AND password = '" + pass + "'"
Access Vulnerable System
Secure Node

Secure Login Page

This system uses parameterized SQL execution (prepared statements). The user inputs are treated strictly as parameter values, preventing SQL engines from executing them as commands.

Secure Code: db.execute("SELECT * FROM users WHERE username = ? AND password = ?", (user, pass))
