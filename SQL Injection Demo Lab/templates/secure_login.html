    Why Parameterization Works

    Instead of merging user strings with raw commands, a parameterized query divides execution into two phases:

    1. Compilation Phase: The SQL statement SELECT * FROM users WHERE username = ? AND password = ? is sent to the DB driver and pre-compiled as a structured blueprint.
    2. Execution Phase: The driver passes input variables directly into the placeholders (?). Because the compilation step has already finished, the engine treats parameter data as literal strings, not instructions.

    Try the same SQL Injection attacks:

    Input ' OR 1=1 -- in the username box. Notice that instead of letting you bypass credentials, the system outputs "Invalid username or password".

    The SQLite query engine literally searches the database for a user whose actual name is ' OR 1=1 --. Since that doesn't exist, the login fails safely.
